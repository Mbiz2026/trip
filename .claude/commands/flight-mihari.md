# 航空券見張り番 (/flight-mihari)

`travel_tools/flight_finder.html`(航空券サーチ司令塔)の2階部分。アプリ単体ではできない「毎日チェックして通知」をClaudeの定期実行(Routine)で代行する。

## 入力の読み取り

ユーザー入力から次を読み取る。足りない項目だけ聞き返す(全部そろっていれば確認のみで開始):

- **モード**: `価格`(特定路線の7日間追跡)か `セール`(期間内の日本発セール・エラーフェア監視)
- 価格モード: 路線(例: KIX→BKK)・出発日・帰国日(片道なら無し)・**基準額**(検索時に見つけた最安値。円)
- セールモード: 監視期間(例: 2026-08-01〜08-31)・出発地(既定: 関西発。伊丹→関空→神戸→羽田の優先順)・方面(おまかせ可)
- 通知手段の希望(既定: プッシュ通知。使えない場合はセッション内報告のみになる旨を伝える)

## 大事な前提(ユーザーに必ず伝える)

1. **ベストエフォートである**: この実行環境は外部サイトへの直接アクセスが制限されており(多くのサイトが403)、価格の取得は主に**Web検索の結果(検索スニペット・相場ページ)**に頼る。正確な実勢価格が取れない日は「取得失敗」と正直に報告する。**嘘の価格をでっち上げることは絶対にしない。**
2. セール監視はニュース検索ベースなので価格追跡より安定して動く。
3. 頻度は**1日1回まで**。サイトへの過剰アクセスはしない。
4. 「やめて」と言われたら `delete_trigger` で即座に停止する。

## 手順

### 1. 初回チェック(その場で1回目を実行)

- 価格モード: WebSearchで現在の相場感を調べる(例: `KIX BKK 航空券 12月 最安値`、`site:kayak.co.jp KIX BKK` 等)。Googleフライト/Skyscanner/Kayakの相場ページのスニペットから価格帯を拾う。取れたら基準額と比較して報告。
- セールモード: WebSearchで直近のセール・エラーフェアを掃く(例: `航空券 セール 関西発`、`エラーフェア 日本発`、`site:traicy.com セール`、`site:sky-budget.com セール`、`AirAsia BIG SALE`、`Peach セール` 等)。期間・出発地に合うものだけ報告。

### 2. Routine(定期実行)を作る

- `mcp__claude-code-remote__create_trigger` を使う。**このセッション自身に発火**させる(persistent_session_id等は指定しない)
- `cron_expression`: `0 22 * * *`(UTC 22:00 = 日本時間 朝7:00)
- `name`: `flight-mihari-<路線または期間>`(既存の見張りは `list_triggers` で確認し、重複を作らない)
- `prompt` に以下を**すべて埋め込む**(発火時にこの指示だけで動けるように): モード/路線・日付/基準額/監視期間/開始日/終了日(開始から7日後)/通知手段/このファイルの手順に従うこと

### 3. 毎日の発火時にやること(Routineのpromptに含める内容)

1. 今日が終了日を過ぎていたら → **最終まとめ**(下記5)へ
2. 手順1と同じ方法でチェックを実行
3. 判定:
   - 価格モード: 確認できた最安の目安が**基準額を下回った** → 通知
   - セールモード: **前日までに報告していない新しいセール・エラーフェア**が期間・出発地に合致 → 通知(何を既報としたかはセッションの文脈で覚えている)
   - 取得失敗 → その日は静かに記録。**2日連続で失敗したら**その旨を1回だけ通知
   - 何もなし → 通知しない(静かに1行記録)
4. 通知の出し方: ToolSearchで `PushNotification` を読み込んで送る(タイトル例: `✈ KIX→BKK 42,800円(基準45,000円を下回りました)`)。本文には「どこで見つけたか・確認用の検索リンク・次にやること(公式で総額確認→予約)」を含める。PushNotificationが使えなければセッション内に同内容を書き残し、最終まとめで伝える
5. 最終まとめ(7日目または期間終了): 期間中の推移(取れた日の価格・失敗日数・出たセール)を表で報告し、「延長する?」と一言添えて、`delete_trigger` で自分のRoutineを削除する

## 禁止事項

- 取得できなかった日に価格を推測で報告すること(「取得失敗」と書く)
- 1日1回を超える自動アクセス
- 非公式サイト・代行サイトへの誘導(リンクは公式・大手検索サイトのみ)
- ユーザーの承認なしに予約・購入に進むこと(このコマンドは見張りだけ)
