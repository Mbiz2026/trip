# マイル見張り番 (/mile-mihari)

`travel_tools/mile_compass.html`(マイルコンパス)の数字を鮮度切れにしないための定期メンテナンス手順。毎月のRoutineから呼ばれるほか、手動で「/mile-mihari」や「マイルコンパスを最新化して」と頼まれたときも同じ手順で動く。

## 点検対象(優先順)

1. **燃油サーチャージ**(最重要・2ヶ月ごとに改定): JAL/ANAの翌期発券分の発表。偶数月の中旬(6月・8月・10月…)に翌2ヶ月分が発表される。公式プレスリリース(press.jal.co.jp / anahd.co.jp)+報道(Aviation Wire・TRAICY・トラベルWatch等)で確認
2. **マイルチャート改定**: JAL国際線特典PLUS / JAL提携社特典 / ANA国際線特典 / BA Avios / アジアマイル / ヴァージン×ANA。近年ほぼ毎年どこかが改定している(Aviosは予告なし改定の前科あり)
3. **アメックスMR移行先**の追加・終了・レート変更(公式 americanexpress.com/ja-jp + 報道)
4. **ワンワールド加盟**の変動(フィリピン航空の加盟完了が近い。加盟したらマニラ・セブ周りを更新)
5. **就航路線**の大きな変化(関空発を優先。JAL関空5路線・キャセイ・カタール・フィンエアー・マレーシア航空、JAL羽田-ドーハの運休解除など)
6. **「取り方道場」タブの制度情報**: 予約開始日ルール(JAL=360日前0時/ANA=355日前9時)、取消手数料(JAL 3,100円/ANA 3,000マイルor3,000円)、MR→ANA年4万マイル上限、Avios/アジアマイル/カンタスのチャート改定、燃油ゼロ会社リスト(2026年にUAが日本発で徴収開始した類の変化)

## 裏どりの基準(これを満たしたときだけファイルを更新する)

- **確定** = 「公式ソース(プレスリリース/公式ページ)の記載」または「独立した報道2つ以上の一致」。数字・適用開始日・対象範囲の3点が揃っていること
- **未確定** = 検索スニペット1つだけ/情報源同士が食い違う/ブログのみ → **更新せず**、報告に「要確認」として書くだけ
- この環境は公式サイト本文へ直接アクセスできないことが多い(プロキシ403)。検索結果経由になる場合は、独立ソースの一致を必ず取る。**推測で数字を埋めることは絶対にしない**(既存の※印ルールを守る)

## 更新手順(確定情報があったとき)

1. `main` を最新化して作業する(`git fetch origin main && git checkout main && git pull origin main`)
2. 反映箇所は4種類。取りこぼしやすいので全部見る:
   - **行き先カード・一覧表**: `travel_tools/dev/gen_mile_compass_cards.py` の燃油定数(KR/EA_J/…)やD(行き先データ)を編集 → `python3 travel_tools/dev/gen_mile_compass_cards.py` で再生成
   - **「マイル術」タブ**の燃油早見表・MR移行表・スイートスポット(HTML直書き)
   - **「基礎と出典」タブ**のチャート表・「燃油は◯〜◯月発券分」の文言(HTML直書き)
   - **確認日スタンプ**: ヘッダーの「情報確認日」・フッター・出典の日付を点検日に更新
3. **表示確認**(CLAUDE.mdルール5): playwrightで JSあり/なし × ライト/ダーク のスクリーンショットを撮って崩れがないか確認
4. `main` に直接コミット&プッシュ(数値・日付の差し替えに限る。ユーザーが自動更新を承認済み: 2026-07-16)。コミットメッセージに「何が・いつから・出典」を書く
5. **例外**: チャートの構造変更(ゾーン制→路線別のような大改定)やアプリの作り自体に関わる変更は、mainに直接入れず**ブランチ+プルリクエスト**にして通知で相談する

## 報告のしかた

- **更新あり**: 何をどう変えたか(旧→新)と出典を短く報告。プッシュ通知が使えるなら送る(例: `⛽ 燃油改定: 9-10月発券分は欧米¥52,000に値下がり → マイルコンパス更新済み`)
- **変更なし**: 静かに終了(通知しない)。
- **取得失敗・未確定のみ**: 正直にその旨だけ報告。でっち上げない
- `travel_tools/mile_compass.html` が main に存在しない(未マージ)場合: 何も変更せず、その旨だけ報告して終了

## 禁止事項

- 裏どり基準を満たさない数字での更新
- 確認日スタンプだけ進めて中身を確認しない「空更新」
- 非公式サイト・代行サイトへのリンク追加(公式と大手報道のみ)
- force push等の破壊的git操作
