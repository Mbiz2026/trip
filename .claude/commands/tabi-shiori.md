# 旅のしおり自動生成

ユーザーの入力(例: `/tabi-shiori ソウル 9/20〜23 大人2人`)から行き先・時期・日数・人数・要望を読み取る。行き先が無い場合だけ聞き返す。

## 手順

1. **調査**(WebSearch / WebFetch を使用。各項目に出典URLと確認日を必ず控える)
   - 入国手続き: 電子認証・入国カードの要否と公式申請URL(政府公式のみ。代行サイトを案内しない)
   - 外務省 海外安全ホームページの危険情報・スポット情報(楽観的に言い換えない)
   - その時期の気候・服装の目安、日の入り時刻
   - 空港→市内の移動手段と料金相場、定番の交通アプリ
   - 通貨・両替・チップ・カード事情(現金がどれくらい必要か)
   - eSIM・通信事情
   - その時期の祝日・イベント・休業日
   - その土地の定番の詐欺・ぼったくり手口と回避策
   - モデルコースやグルメは要望があれば追加

2. **生成**: `travel_tools/shiori_<ローマ字地名>_<出発日YYYYMMDD>.html` に1ファイルで作る
   - デザインは `travel_tools/kaigai_travel_kit.html` のデザイントークン(色・フォント・タブ構成)を流用し、姉妹アプリに見えるようにする
   - 外部リソース(CDN・画像URL・Webフォント)は読み込まない。オフラインで動くこと
   - **CLAUDE.mdの絶対ルールに従い、タブ切り替え・情報の閲覧はCSSだけ(ラジオボタン+`:checked`)で動く構造にする。** `kaigai_travel_kit.html` と同じパターン(`.tabradio`/`.langradio` + `~` セレクタ)をそのまま流用してよい。JavaScriptは計算機・検索・保存など「あれば便利」機能だけに限定する
   - ヘッダーに「情報確認日」を明記し、フッターに「渡航前に公式サイトで再確認」の免責を入れる
   - 不確かな情報は本文中に「要確認」と明記する

3. **検証**: ヘッドレスブラウザで表示確認する(セッション開始時のフックが利用可否とパスを教えてくれる)
   ```
   <ブラウザパス> --no-sandbox --disable-gpu --window-size=390,1500 --screenshot=/tmp/shiori.png "file://<絶対パス>"
   ```
   **JSを外した状態も必ず確認する**(`--blink-settings=scriptEnabled=false` を追加。iOSファイルアプリのプレビュー相当)。タブ切り替え・全情報の閲覧がJS無しで動くことを確認してから納品する。ヘッドレスブラウザが無い環境ではユーザーに確認を依頼する

4. **納品**: Artifactとして公開し、HTMLファイルも送付。要点3行+リンク+「オフラインで使うにはファイルを保存」の案内を日本語で報告する。

## 禁止事項

- 出典の無い断定(特に治安・料金・営業時間)
- 政府公式以外の申請サイトへの誘導
- ユーザーの個人情報をしおりに埋め込むこと(緊急メモ欄は空欄で用意する)
- JavaScriptが無いと閲覧・タブ切り替えができない構造にすること
