# 設計まとめ: 旅先えらび(仮称)

作成日: 2026-07-12 / 状態: **ユーザー承認待ち(実装前)**

「次の海外旅行先が、候補が多すぎて決められない」を解決するツール。
行動経済学・心理学の知見をもとに、ユーザーへのヒアリングで決めた土台。

---

## 1. ヒアリングで決まったこと

| 項目 | 決定内容 |
|---|---|
| 解決する困りごと | 候補が多すぎて絞れない(選択過負荷) |
| 使う人 | まずは自分ひとり用。将来は誰でも使える形に |
| 納得のスタイル | 「十分条件を満たせばOK」型(満足化。研究上、選択後の幸福度が高い傾向) |
| ゴール(出口) | 納得できる2〜3候補まで絞る。最終決定は自分で |
| 絞り込み方式 | 条件を自分で動かして絞る検索型 |
| 候補の範囲 | 日本発の定番海外 約40都市 |
| 判断軸 | ①移動の負担(飛行時間・時差) ②お金(予算感・物価) ③時期(気候・ベストシーズン) |
| 納得感の仕掛け | 外した理由の見える化/残った候補のトレードオフ表 |
| 残った候補の見せ方 | アプリは順位付けしない。全部そのまま見せて自分で見比べる |

## 2. 行動経済学的な設計原則(なぜこの形か)

1. **選択過負荷への対処** — 40都市を一度に見比べさせない。条件で消していき、残りが見渡せる数になってから中身を見る。
2. **属性による消去(elimination by aspects)** — 人が自然にやる「◯◯は絶対ムリ、だから除外」という思考の流れをそのままUIにする。条件は段階式ボタン(〜5時間/〜9時間/制限なし など)。
3. **後悔回避** — 外れた候補は消さずに「なぜ外れたか」の理由バッジ付きで折りたたみ表示。「あっちはそもそも条件外だった」と確認できると、決めた後の後悔が減る。
4. **並列比較でトレードオフを見える化** — 残った候補を同じ軸で並べた比較表を用意。「近いけど高い/安いけど遠い」の構造が見えると選びやすい。
5. **自律性の尊重** — アプリはおすすめ順に並べ替えない(順位付けしない)。「自分で最終決定した」感覚が満足度を上げるため、最後のひと押しはユーザーに残す。
6. **満足化のフレーミング** — 画面の文言は「ベストを探す」ではなく「あなたの条件を満たす候補はこれだけ」という枠組みで統一する。

## 3. 画面構成(1ページ・スマホ縦向き基準)

1. **条件パネル**(上部・常に見える)
   - 移動の負担: 直行便の飛行時間で「〜5時間/〜9時間/制限なし」+時差の表示
   - お金: 旅全体の予算目安で「〜10万円/〜20万円/〜35万円/制限なし」
   - 時期: 行く月を1〜12月から選択 → ベストシーズン外(雨季・酷暑・厳寒)の候補を除外
   - 各条件は段階式ボタン(ラジオボタン)。**JavaScriptなし・CSSだけで絞り込みが動く**
2. **残った候補ゾーン** — 「残り◯都市」と表示し、都市カードを並べる(順不同=地域順の固定)。カードには飛行時間・時差・予算目安・ベストシーズン・ひとこと特徴。
3. **外れた候補ゾーン** — 折りたたみ表示。各都市に「飛行時間オーバー」「予算オーバー」「その月は雨季」などの理由バッジ。
4. **トレードオフ比較表** — 残った候補を同じ軸(飛行時間/時差/予算/シーズン)で横並びにした表。気になる都市にチェックを付けると行がハイライトされる(これもCSSのみで動作)。

## 4. 内蔵データの方針(正確性のルール)

- **安定情報**(飛行時間の目安・時差・ベストシーズンの一般傾向): アプリに内蔵。出典と確認日を明記。
- **変動情報**(予算・物価の目安): 「◯年◯月時点の目安」と確認日を必ず付け、幅のあるレンジで表記。断定しない。
- **制度・治安**(ビザ・危険情報): このアプリでは扱わず、外務省 海外安全ホームページ等の**公式リンクの案内のみ**とする(既存の kaigai_travel_kit.html の役割と分担)。

### 候補40都市の案(実装前に調整可)

- 東アジア: ソウル/釜山/台北/高雄/香港/マカオ/上海
- 東南アジア: バンコク/チェンマイ/シンガポール/クアラルンプール/ハノイ/ダナン/ホーチミン/セブ/バリ島/プーケット/プノンペン(シェムリアップ)
- 南アジア・中東: デリー/ドバイ/イスタンブール
- 太平洋: グアム/サイパン/ホノルル
- オセアニア: シドニー/メルボルン/ケアンズ/ゴールドコースト/オークランド
- 北米: ロサンゼルス/サンフランシスコ/ラスベガス/ニューヨーク/バンクーバー
- 欧州: ロンドン/パリ/ローマ/バルセロナ/ウィーン/プラハ/ヘルシンキ

## 5. 技術方針(リポジトリの絶対ルール準拠)

- 1ファイルHTML(`travel_tools/tabisaki_erabi.html` 予定)。外部リソース(CDN・Webフォント・外部画像)なし、オフラインで動作。
- 絞り込み・折りたたみ・比較表ハイライトはすべてCSSのみで動作(ラジオ/チェックボックス+`:checked`)。JavaScriptは使うとしても補助機能のみ。
- ライト/ダークテーマ両対応。デザインは kaigai_travel_kit.html と統一感を持たせる。
- 納品前にヘッドレスブラウザでJSあり/なし両方のスクリーンショット確認。
